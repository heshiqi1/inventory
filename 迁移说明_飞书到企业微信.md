# 从飞书迁移到企业微信说明

## 一、主要改动汇总

### 1. 配置变量名称

| 原飞书版本 | 新企业微信版本 |
|-----------|---------------|
| `FEISHU_WEBHOOK_URL` | `WECOM_WEBHOOK_URL` |

### 2. 函数名称变更

| 原函数名 | 新函数名 | 说明 |
|---------|---------|------|
| `is_flow_webhook()` | 已删除 | 企业微信无需区分类型 |
| `send_feishu()` | `send_wecom_markdown()` | 发送 Markdown 消息 |
| `send_feishu_text()` | `send_wecom_text()` | 发送文本消息 |
| `format_signal_card()` | 已删除 | 飞书交互式卡片 |
| `format_signal_for_flow()` | 已删除 | 飞书流程格式 |
| - | `format_signal_markdown()` | 新增企业微信 Markdown |

### 3. 消息格式变化

#### 飞书版本（交互式卡片）
```python
{
    "msg_type": "interactive",
    "card": {
        "header": {...},
        "elements": [...]
    }
}
```

#### 企业微信版本（Markdown）
```python
{
    "msgtype": "markdown",
    "markdown": {
        "content": "## 标题\n> 内容..."
    }
}
```

### 4. API 返回值差异

| 平台 | 成功标志 | 失败信息字段 |
|-----|---------|-------------|
| 飞书 | `code == 0` | `msg` |
| 企业微信 | `errcode == 0` | `errmsg` |

## 六、常见问题

### Q1: 为什么企业微信版本的消息没有飞书那么美观？

**答：** 企业微信的 Markdown 支持有限，但核心信息展示完全相同。如果需要更美观的格式，可以考虑：
- 使用更多的 Emoji 图标
- 合理使用颜色标签
- 优化文本排版

### Q2: 企业微信版本能发送图片吗？

**答：** 企业微信机器人支持图片，但需要图片 URL。可以扩展代码添加图片功能：

```python
payload = {
    "msgtype": "image",
    "image": {
        "base64": "图片base64编码",
        "md5": "图片MD5值"
    }
}
```

### Q3: 可以同时支持飞书和企业微信吗？

**答：** 可以！只需配置两个 Webhook 地址，发送时调用两个函数即可：

```python
FEISHU_WEBHOOK_URL = "..."
WECOM_WEBHOOK_URL = "..."

# 发送信号
send_feishu(feishu_format)
send_wecom_markdown(wecom_format)
```

### Q4: 企业微信版本的代码是否兼容飞书？

**答：** 不兼容。两个平台的 API 完全不同。但你可以保留原 `forex_monitor_feishu.py` 文件，同时使用新版本。
