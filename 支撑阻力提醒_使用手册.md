# 支撑阻力提醒系统 — 使用手册

基于 **Twelve Data** 的 K 线数据，对多品种进行支撑/阻力与趋势分析，当价格接近关键位时通过**企业微信**推送通知；支持**关键位 + 反转形态**的重点提醒、本地 K 线图绘制与多 API Key 自动切换。

---

## 一、功能说明

| 功能 | 说明 |
|------|------|
| 数据来源 | Twelve Data API，按配置的 K 线周期拉取最新约 200 根 K 线 |
| 分析内容 | 趋势（EMA20/50/100）、上升/下降趋势线、通道线、水平支撑/阻力、斐波那契 50%/61.8% 回调、最近波段高低点、反转形态（锤子线、看涨/看跌吞没） |
| 通知条件 | 仅在看涨或看跌趋势下，当价格**接近**某支撑/阻力位（默认 0.1% 以内）时发送通知；中性趋势不通知 |
| 重点通知 | **关键位置附近出现反转形态**时，使用「🚨 重点通知」标题并强调需重点关注 |
| 本地图表 | 每次触发通知前，在 `monitor_data/charts/` 下保存带趋势线、通道线、斐波位、当前价与关键价标注的 K 线图；可选是否随通知发送到企业微信 |
| 多 Key 切换 | 支持配置多个 Twelve Data API Key，某 Key 被限流或额度用尽时自动切换到下一个 |
| 本地记录 | 监控摘要与触发记录保存在 `monitor_data/`，可追溯历史 |

---

## 二、环境与依赖

- **Python**：3.7+
- **依赖**：`requests`、`pandas`、`numpy`；若需本地绘制 K 线图需 `matplotlib`

安装依赖示例：

```bash
pip install requests pandas numpy matplotlib
```

## 三、配置说明

所有配置在 **支撑阻力提醒.py** 文件顶部「配置参数」区域修改。

### 3.1 Twelve Data API Key（必配）

支持**多个 Key**，被限流或额度用尽时自动切换。

**方式一：环境变量（推荐）**

- 单个 Key：`TWELVEDATA_API_KEY=你的Key`
- 多个 Key：`TWELVEDATA_API_KEY=key1,key2,key3`（英文逗号分隔）

Windows CMD：

```cmd
set TWELVEDATA_API_KEY=你的Key
```

Windows PowerShell：

```powershell
$env:TWELVEDATA_API_KEY="key1,key2,key3"
```

**方式二：脚本内直接写**

若未设置环境变量，则使用脚本中的 `TWELVEDATA_API_KEYS` 列表，例如：

```python
TWELVEDATA_API_KEYS = ["key1", "key2", "key3"]
```

### 3.2 监控品种

在 `SYMBOLS` 中配置，格式为 `{'内部符号': 'TwelveData符号'}`：

```python
SYMBOLS = {
    'USDJPY': 'USD/JPY',
    'EURUSD': 'EUR/USD',
    'XAUUSD': 'XAU/USD',
    'GBPUSD': 'GBP/USD',
    # 'BTCUSD': 'BTC/USD',   # 取消注释即可启用
}
```

增删或注释某行即可调整监控品种。

### 3.3 分析周期与监控间隔

| 配置项 | 含义 | 默认值 | 可选示例 |
|--------|------|--------|----------|
| `ANALYSIS_TIMEFRAME` | 分析使用的 K 线周期 | `'15min'` | `'5min'`, `'15min'`, `'1h'`, `'4h'` 等（Twelve Data 支持的 interval） |
| `TIMEFRAMES` | 拉取数据的周期列表 | `[ANALYSIS_TIMEFRAME]` | 一般与上面保持一致即可 |
| `MONITOR_INTERVAL_SECONDS` | 每轮全品种检查完后，等待多少秒再下一轮 | `300` | `60`（1 分钟）、`300`（5 分钟） |

示例：用 5 分钟 K 线、每 2 分钟检查一轮：

```python
ANALYSIS_TIMEFRAME = '5min'
TIMEFRAMES = [ANALYSIS_TIMEFRAME]
MONITOR_INTERVAL_SECONDS = 120
```

### 3.4 企业微信通知

| 配置项 | 含义 | 默认值 |
|--------|------|--------|
| `WEWORK_WEBHOOK_URL` | 企业微信群机器人 Webhook 地址 | 需改为你自己的地址 |
| `WEWORK_SEND_CHART_IMAGE` | 通知时是否一并发送 K 线图 | `False`（仅文字）；改为 `True` 则发图（图片需 ≤2MB） |

获取 Webhook：企业微信 → 群设置 → 群机器人 → 添加机器人 → 复制 Webhook 地址。

### 3.5 分析阈值与冷却（可选调）

| 配置项 | 含义 | 默认值 |
|--------|------|--------|
| `NEAR_THRESHOLD` | 价格与关键位的“接近”比例，小于该值才提醒 | `0.001`（0.1%） |
| `COOLDOWN_SECONDS` | 同一关键位重复提醒的冷却时间（秒） | `900`（15 分钟） |
| `LEVEL_MERGE_THRESHOLD` | 水平位合并阈值 | `0.001` |
| `EMA_PERIODS` | EMA 周期列表 | `[20, 50, 100]` |

### 3.6 本地存储

| 配置项 | 含义 | 默认值 |
|--------|------|--------|
| `MONITOR_DATA_DIR` | 监控数据根目录 | `"monitor_data"` |
| `MONITOR_CHARTS_DIR` | K 线图保存目录 | `monitor_data/charts` |
| `DATA_RETENTION_DAYS` | 本地记录保留天数 | `30` |

## 四、运行方式

在项目目录下执行：

```bash
python 支撑阻力提醒.py
```

- 启动后会发送一次**企业微信启动通知**（若已配置 Webhook），并拉取各品种初始数据。
- 之后按 `MONITOR_INTERVAL_SECONDS` 间隔循环：每轮遍历所有品种做支撑/阻力判断，满足条件则发通知并可选发图。
- 控制台会持续输出每轮检查结果；**关键位 + 反转形态**时会有 `【重点】关键位+反转形态` 前缀。
- 退出：在运行窗口按 **Ctrl+C**。

## 五、通知类型说明

### 5.1 普通机会通知

当**趋势为看涨或看跌**且**价格接近某支撑/阻力位**时发送，内容包括：

- 品种、当前价格、趋势方向、分析周期
- 接近的支撑位/阻力位及距离百分比
- 若检测到反转形态，会单独列出

标题为：**📊 支撑阻力机会通知**。

### 5.2 重点通知（关键位 + 反转形态）

当**同时满足**以下两点时，会以**重点通知**形式发送：

1. 价格接近至少一个关键支撑/阻力位  
2. 最近 K 线出现反转形态（锤子线、看涨吞没、看跌吞没等）

表现：

- 企业微信标题：**🚨 重点通知：关键位置附近出现反转形态**，并带有“此信号需重点关注”的说明。
- 控制台该条日志前会带 **【重点】关键位+反转形态** 前缀。

### 5.3 不通知的情况

- **中性趋势**：即使有关键位接近，也只写本地记录，不发送企业微信。
- **冷却期内**：同一关键位在 `COOLDOWN_SECONDS` 内不会重复提醒。

## 六、本地图表与数据

### 6.1 K 线图（通知前自动生成）

每次**即将发送机会通知**时，会先在本地生成一张 K 线图，包含：

- K 线（蜡烛图）、EMA20/50/100
- 上升/下降趋势线、通道线
- 最近波段最高/最低值、斐波那契 50% 与 61.8% 回调位
- 当前价格线及关键价位标注

保存路径：`monitor_data/charts/{品种}_{时间}.png`，例如 `GBPUSD_20250305_153022.png`。

是否随通知发送到企业微信由 **`WEWORK_SEND_CHART_IMAGE`** 控制（`True` 发送，图片需 ≤2MB）。

### 6.2 监控记录与摘要

- **monitor_records.json**：每次“有接近关键位且发送了通知”的详细记录（时间、品种、价格、趋势、接近的位置、反转形态等）。
- **{品种}_summary.json**：各品种的监控摘要（含统计与接近位置），即使未发通知也会按轮次写入。

以上文件均在 `monitor_data/` 下，保留天数由 `DATA_RETENTION_DAYS` 控制。

## 七、常见问题

### 7.1 收不到通知

- 确认 **WEWORK_WEBHOOK_URL** 已正确配置且机器人仍在群里。
- 确认当前趋势为**看涨或看跌**（中性不通知）。
- 确认价格确实**接近**某关键位（在 `NEAR_THRESHOLD` 范围内）。

### 7.2 数据加载失败 / API 额度用完

- 免费版 Twelve Data 有每日请求限制；可配置 **多个 API Key**（环境变量逗号分隔或脚本内列表），程序会在限流或额度用尽时自动切换。
- 启动时会打印「已加载 N 个 API Key」；若所有 Key 均不可用，控制台会提示。

### 7.3 企业微信收不到图片

- 将 **WEWORK_SEND_CHART_IMAGE** 设为 `True`。
- 企业微信要求图片不超过 **2MB**；若超限，程序会跳过发图并在控制台提示，可适当减小图表 dpi 或尺寸（需改脚本内绘图参数）。

### 7.4 想更频繁 / 更少提醒

- 更频繁：减小 **MONITOR_INTERVAL_SECONDS**（如 60）；或适当增大 **NEAR_THRESHOLD**（如 0.002）使“接近”更容易触发。
- 更少：增大 **MONITOR_INTERVAL_SECONDS** 或 **COOLDOWN_SECONDS**；或减小 **NEAR_THRESHOLD**。

### 7.5 使用其他 K 线周期

- 修改 **ANALYSIS_TIMEFRAME** 与 **TIMEFRAMES** 即可，例如改为 `'5min'` 或 `'1h'`；通知和图表中的“分析周期”会随之更新。

## 八、配置速查表

| 项目 | 配置变量 | 典型值 |
|------|----------|--------|
| API Key（多 Key） | 环境变量 `TWELVEDATA_API_KEY` 或脚本内 `TWELVEDATA_API_KEYS` | `key1,key2,key3` |
| 监控品种 | `SYMBOLS` | `{'EURUSD':'EUR/USD', ...}` |
| 分析周期 | `ANALYSIS_TIMEFRAME` | `'15min'` / `'5min'` / `'1h'` |
| 每轮监控间隔 | `MONITOR_INTERVAL_SECONDS` | `60` / `300` |
| 企业微信 Webhook | `WEWORK_WEBHOOK_URL` | 机器人完整 URL |
| 通知是否带图 | `WEWORK_SEND_CHART_IMAGE` | `True` / `False` |
| 接近阈值 | `NEAR_THRESHOLD` | `0.001`（0.1%） |
| 同位置冷却 | `COOLDOWN_SECONDS` | `900`（秒） |

按需修改上述配置后，保存脚本并重新运行 `python 支撑阻力提醒.py` 即可生效。
